Antes de compilar ou executar o sistema, certifique-se de que os seguintes componentes estão instalados na máquina:

ComponenteVersão mínimaComo verificar

02

Estrutura do Projeto

Todos os arquivos-fonte estão dentro da pasta java/. Os arquivos de teste ficam na raiz do projeto:

tp2/
  ├── java/
  │   ├── Block.java         // Estrutura de bloco de memória
  │   ├── PartitionMemory.java // Partições Variáveis (Worst-Fit / Circular-Fit)
  │   ├── BuddyMemory.java    // Sistema Buddy
  │   └── Main.java         // Ponto de entrada e leitura de arquivo
  ├── exemplo_tabela1.txt    // Caso de teste – Partições Variáveis
  └── exemplo_visualizacao.txt // Caso de teste – Sistema Buddy
03

Como Compilar

A compilação gera os arquivos .class na mesma pasta java/. Siga os passos abaixo:

  1. Abra o terminal na pasta do projeto 
    cd C:\Users\SeuUsuario\Desktop\TP2_GerenciamentoMemoria\tp2
  2. Entre na pasta java/ 
    cd java
  3. Execute o compilador 
    javac Block.java PartitionMemory.java BuddyMemory.java Main.java

    Se não houver erros, nenhuma saída é exibida — isso é esperado. Os arquivos .class serão criados na mesma pasta.

⚠ A compilação deve ser feita de dentro da pasta java/. Executar o javac de outra pasta causará erro de arquivo não encontrado.
04

Como Executar

Após a compilação, execute o programa com o comando abaixo, a partir da pasta java/:

java Main

O programa iniciará no modo interativo e guiará o usuário por todas as escolhas necessárias.

05

Fluxo de Interação

O programa realiza perguntas em sequência. Abaixo está o fluxo completo:

  1. Tamanho da memória principal

    Deve ser uma potência de 2 (ex.: 32, 64, 128, 256…). Valores inválidos são rejeitados com mensagem de erro e a pergunta é repetida.

  2. Tipo de gerenciamento

    Digite 1 para Partições Variáveis ou 2 para Sistema Buddy. Qualquer outro valor é rejeitado.

  3. Política de alocação (somente se escolher Partições Variáveis)

    Digite 1 para Worst-Fit (maior bloco disponível) ou 2 para Circular-Fit (busca circular a partir do último ponto).

  4. Caminho do arquivo de requisições

    Informe o caminho para o arquivo .txt com os comandos. Se estiver na pasta java/, use caminho relativo:

    ..\exemplo_tabela1.txt
06

Arquivo de Requisições

O arquivo de entrada contém um comando por linha. Dois tipos são aceitos:

ComandoDescriçãoExemplo
IN(id, tamanho)Aloca um processo com o ID e tamanho indicadosIN(A, 10)
OUT(id)Libera o processo com o ID indicadoOUT(A)

Linhas em branco e linhas mal formatadas são ignoradas silenciosamente. O ID pode ser qualquer sequência de caracteres sem espaço. O tamanho deve ser um inteiro maior que zero.

Exemplo de arquivo válido
IN(A, 10)
IN(B, 15)
IN(C, 23)
OUT(A)
OUT(B)
IN(D, 13)
OUT(D)

10

Solução de Problemas

ProblemaSolução
java não reconhecido no terminal Instale o JDK e certifique-se de que a pasta bin/ do JDK está no PATH do sistema
error: file not found: Block.java Execute o javac de dentro da pasta java/, não da raiz do projeto
Arquivo de requisições não encontrado Use ..\exemplo_tabela1.txt (Windows) ou ../exemplo_tabela1.txt (Linux/Mac) — o ..\ sobe um nível da pasta java/
Acentos aparecem como ? no terminal Use o terminal integrado do VS Code ou adicione -Dstdout.encoding=UTF-8 ao comando: java -Dstdout.encoding=UTF-8 Main
Resultado diferente do esperado com Circular-Fit O cursor circular persiste entre alocações — o próximo bloco pesquisado começa após o último alocado, não desde o início
